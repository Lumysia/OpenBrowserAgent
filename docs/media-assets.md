# Media Assets Guide

Screenshots and short clips are useful for release credibility, but they should be factual and not overpromise automation reliability.

## Recommended README Assets

Use screenshots first. Add a video after the release candidate is stable.

Recommended screenshots:

- Sidepanel chat with a streaming answer and visible tool card.
- Page/context attachment menu with tabs, files, skills, and element selection.
- Rich Markdown output with code, citations, image cards, math, or Mermaid preview.
- Providers settings with model selection.
- Agents or Skills settings showing customization.
- MCP settings with a tested server and tool toggles.

## Optional Demo Video

A 60-90 second video is enough:

1. Open the sidepanel on a web page.
2. Attach the current tab or selected element.
3. Ask a question about the page.
4. Show a tool card or citation.
5. Switch briefly to Providers or Agents settings.
6. End with chat history or export.

Avoid recording private API keys, private pages, browser profile data, or personal accounts.

## Suggested File Layout

If media assets are added, keep them under:

```text
docs/assets/
```

Recommended filenames:

- `sidepanel-chat.png`
- `context-menu.png`
- `tool-card.png`
- `providers-settings.png`
- `agents-settings.png`
- `demo-short.mp4` or `demo-short.gif`

Prefer optimized images and short clips so the README remains fast to load.
